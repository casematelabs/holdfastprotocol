The Problem

AI agents are executing transactions, managing funds, and settling contracts without human intervention. But the infrastructure they run on was never designed for this: agents authenticate with software keys in environment variables, there's no verifiable on-chain identity, no track record, and no programmable settlement when things go wrong.

"21,000 agents entered the Colosseum AI hackathon in February 2026. The accountability layer doesn't exist yet."

Three Composable Primitives

01 · Identity

Hardware-attested agent identities on Solana. Agents register a secp256r1 (P-256) keypair — the same curve as Apple Secure Enclave and FIDO2 hardware keys — anchored on-chain via Solana's native precompile (SIMD-48). Not a software key in an .env file.

02 · Reputation

On-chain reputation oracle. Score: 0–10,000 bp (5,000 = neutral). Updated by the escrow program — not a human — after every pact outcome. Decays lazily when inactive. Readable by any Solana program via a single CPI account read. No oracle fee, no bridge.

03 · Escrow

Programmable settlement for agent commerce. Task-based, milestone-gated, or time-locked release conditions. Funds lock at creation. Release opens a 7-day dispute window with arbiter resolution and on-chain finality. Reputation-gated: bad counterparties fail pre-flight before fees are paid.

Why Solana, Not EVM?

EVM / EAS
  • Cross-contract attestation calls
  • EVM gas per verification
  • 12-second block time
  • Bridge required for non-EVM agents
  • General-purpose, not agent-native
Holdfast on Solana
  • Instruction-level secp256r1 (SIMD-48)
  • Sub-cent per operation
  • ~400ms finality
  • CPI-native composability, day one
  • Purpose-built for AI agent economies
