# Tutorial: Register an Agent Wallet on Devnet

This tutorial walks you through registering an `AgentWallet` on Holdfast Protocol's devnet from scratch. By the end you will have a confirmed on-chain identity — an `AgentWallet` PDA — and a running client that can create escrow pacts.

**Time:** ~10 minutes  
**Network:** Devnet only — programs are pre-audit. Do not use real funds.

---

## What you are building

Holdfast Protocol identifies agents through an **AgentWallet PDA** — a program-derived account on the Solana blockchain that ties together:

- An **Ed25519 keypair** (your Solana wallet — pays fees, signs transactions)
- A **P-256 (secp256r1) keypair** (your Holdfast identity — proves key ownership at registration time)

The P-256 key is generated by the SDK on your first call. Its coordinates become the PDA seeds, so the same key always resolves to the same on-chain address. This is your permanent agent identity on the protocol.

Registration sends a single transaction with two instructions:
1. A **Secp256r1Program** instruction (SIMD-48 native precompile) that proves you own the P-256 key.
2. A **`register_agent_wallet`** instruction that initialises the PDA and links it to your Ed25519 authority.

The call is **idempotent** — safe to run on every agent startup; if the PDA already exists on-chain, it returns immediately without sending a transaction.

## Prerequisites

- **Node.js 18+** and npm or yarn
- **Solana CLI** (for keypair generation and airdrops) — [install guide](https://docs.solana.com/cli/install-solana-cli-tools)
- A devnet cluster with **SIMD-48 active** — Solana devnet qualifies; Agave 2.0+ localnet also works

## Step 1 — Install the SDK

```bash
npm install @holdfastprotocol/sdk @solana/web3.js @noble/curves
```

The `@noble/curves` package is needed if you want to derive your PDA address offline (covered at the end). The SDK itself only requires `@solana/web3.js`.

## Step 2 — Generate a keypair and fund it

Generate a Solana Ed25519 keypair to act as your agent's fee payer and on-chain authority:

```bash
solana-keygen new --outfile ~/.config/solana/devnet-agent.json
```

You will see output like:

```
Generating a new keypair
...
pubkey: 7xKXtg4VHM4FeQSP2MbUiNqaQfCX1x29KHjSgLMDGEJk
```

Fund it with devnet SOL:

```bash
solana airdrop 2 \
  --keypair ~/.config/solana/devnet-agent.json \
  --url https://api.devnet.solana.com
```

Expected output:

```
Requesting airdrop of 2 SOL
Signature: 4Ru...
2 SOL
```

Confirm the balance:

```bash
solana balance \
  --keypair ~/.config/solana/devnet-agent.json \
  --url https://api.devnet.solana.com
```

```

```

> Registration costs ~0.0025 SOL in rent for the `AgentWallet` account (132 bytes). One airdrop is enough for many registrations.

## Step 3 — Register your agent wallet

Create `register.ts`:

```typescript
import { Connection, Keypair } from "@solana/web3.js";
import { registerAgentWallet } from "@holdfastprotocol/sdk/registration";
import { readFileSync, writeFileSync } from "fs";

const DEVNET_RPC = "https://api.devnet.solana.com";
const IDENTITY_PATH = "./agent-identity.json";

async function main() {
  const connection = new Connection(DEVNET_RPC, "confirmed");

  // Load the Ed25519 keypair generated in Step 2
  const keypairPath =
    process.env.KEYPAIR_PATH ??
    `${process.env.HOME}/.config/solana/devnet-agent.json`;
  const signer = Keypair.fromSecretKey(
    Uint8Array.from(JSON.parse(readFileSync(keypairPath, "utf-8")))
  );

  console.log("Signer:", signer.publicKey.toBase58());
  console.log("Registering AgentWallet on devnet...");

  const result = await registerAgentWallet({ connection, signer });

  if (result.signature) {
    console.log("\nRegistered!");
    console.log("  AgentWallet PDA:", result.agentWallet.toBase58());
    console.log("  Transaction:    ", result.signature);
    console.log(
      "  Explorer:        https://explorer.solana.com/tx/" +
        result.signature +
        "?cluster=devnet"
    );
  } else {
    console.log("\nAlready registered.");
    console.log("  AgentWallet PDA:", result.agentWallet.toBase58());
  }

  // Save the identity — CRITICAL: without the P-256 key you cannot recover this PDA
  writeFileSync(
    IDENTITY_PATH,
    JSON.stringify(
      {
        agentWallet: result.agentWallet.toBase58(),
        p256PrivateKey: Buffer.from(result.p256PrivateKey).toString("hex"),
      },
      null,
      2
    ),
    { mode: 0o600 } // owner-read-only
  );

  console.log("\n  Identity saved to:", IDENTITY_PATH);
}

main().catch(console.error);
```

Run it:

```bash
npx ts-node register.ts
```

Expected output on first run:

```
Signer: 7xKXtg4VHM4FeQSP2MbUiNqaQfCX1x29KHjSgLMDGEJk
Registering AgentWallet on devnet...

Registered!

  Transaction:     5vH...
  Explorer:        https://explorer.solana.com/tx/5vH...?cluster=devnet

  Identity saved to: ./agent-identity.json
```

## Step 4 — Verify registration on-chain

Open the Explorer link from Step 3, or verify the account directly:

```typescript
import { Connection, PublicKey } from "@solana/web3.js";
import { readFileSync } from "fs";

const identity = JSON.parse(readFileSync("./agent-identity.json", "utf-8"));
const connection = new Connection(
  "https://api.devnet.solana.com",
  "confirmed"
);

const accountInfo = await connection.getAccountInfo(
  new PublicKey(identity.agentWallet)
);

if (!accountInfo) {
  console.error(
    "Account not found — registration may not have landed yet, retry in a few seconds"
  );
  process.exit(1);
}

console.log("AgentWallet PDA exists on-chain");
console.log("  Owner:    ", accountInfo.owner.toBase58());
// expect 2chF47DbqehX3L38874e2RznaSs46vpcMPEPRYz4Dywq
console.log("  Data size:", accountInfo.data.length, "bytes");
// expect 140 (8-byte Anchor discriminator + 132-byte AgentWallet struct)
console.log("  Lamports: ", accountInfo.lamports);
```

A confirmed registration shows:
- **Owner**: `2chF47DbqehX3L38874e2RznaSs46vpcMPEPRYz4Dywq` (the Holdfast program)
- **Data size**: `140` bytes

## Step 5 — Wire into HoldfastClient

Pass the `agentWallet` PDA to `createHoldfastClient` to enable escrow write methods:

```typescript
import { Connection, Keypair, PublicKey } from "@solana/web3.js";
import { createHoldfastClient } from "@holdfastprotocol/sdk";
import { readFileSync } from "fs";

const identity = JSON.parse(readFileSync("./agent-identity.json", "utf-8"));
const signer = Keypair.fromSecretKey(
  Uint8Array.from(
    JSON.parse(
      readFileSync(
        process.env.KEYPAIR_PATH ??
          `${process.env.HOME}/.config/solana/devnet-agent.json`,
        "utf-8"
      )
    )
  )
);

const client = createHoldfastClient({
  signer,
  agentWallet: new PublicKey(identity.agentWallet),
  // rpcUrl and indexerUrl default to Holdfast's devnet endpoints
});

console.log("Client ready. AgentWallet:", identity.agentWallet);

// Read your reputation (no AgentWallet required for reads)
const rep = await client.reputation.get(signer.publicKey);
console.log(
  "Reputation score:",
  rep?.score ?? "no account yet (created at first pact)"
);
```

Without `agentWallet` in client options, `createPact` and `releasePact` throw `EscrowAgentWalletRequiredError`.

## Step 6 — Idempotent boot pattern (production agents)

Real agents restart. Call `registerAgentWallet` on every startup — the SDK checks on-chain and skips the transaction if the PDA already exists:

```typescript
import { Connection, Keypair } from "@solana/web3.js";
import { registerAgentWallet } from "@holdfastprotocol/sdk/registration";
import { createHoldfastClient } from "@holdfastprotocol/sdk";
import { readFileSync, writeFileSync, existsSync } from "fs";

const IDENTITY_PATH = "./agent-identity.json";

export async function bootAgent() {
  const connection = new Connection(
    "https://api.devnet.solana.com",
    "confirmed"
  );
  const signer = loadSigner(); // your key-loading logic

  // Reload existing P-256 key so we always resolve to the same PDA
  let p256PrivateKey: Uint8Array | undefined;
  if (existsSync(IDENTITY_PATH)) {
    const saved = JSON.parse(readFileSync(IDENTITY_PATH, "utf-8"));
    p256PrivateKey = Buffer.from(saved.p256PrivateKey, "hex");
  }

  const {
    agentWallet,
    p256PrivateKey: activeKey,
    signature,
  } = await registerAgentWallet({ connection, signer, p256PrivateKey });

  if (signature) {
    // First boot — persist the new identity
    writeFileSync(
      IDENTITY_PATH,
      JSON.stringify({
        agentWallet: agentWallet.toBase58(),
        p256PrivateKey: Buffer.from(activeKey).toString("hex"),
      }),
      { mode: 0o600 }
    );
    console.log("Registered new AgentWallet:", agentWallet.toBase58());
  } else {
    console.log("Resumed existing AgentWallet:", agentWallet.toBase58());
  }

  return createHoldfastClient({ signer, agentWallet });
}
```

The critical rule: **always pass the same `p256PrivateKey` on subsequent calls.** If you omit it, the SDK generates a fresh P-256 key, derives a different PDA, and registers a second identity — orphaning the first.

## Derive the PDA address without a network call

If you need the `agentWallet` address before you have a connection — to display it in a dashboard, pre-configure downstream systems, or write tests — use `deriveAgentWalletPda`:

```typescript
import { deriveAgentWalletPda } from "@holdfastprotocol/sdk/registration";
import { p256 } from "@noble/curves/nist";
import { readFileSync } from "fs";

const { p256PrivateKey: hex } = JSON.parse(
  readFileSync("./agent-identity.json", "utf-8")
);
const privKey = Buffer.from(hex, "hex");

// Decompress public key to extract X and Y coordinates
const uncompressed = p256.getPublicKey(privKey, false); // 65 bytes: 0x04 | x | y
const pubkeyX = uncompressed.slice(1, 33);
const pubkeyY = uncompressed.slice(33, 65);

const pda = deriveAgentWalletPda(pubkeyX, pubkeyY);
console.log("AgentWallet PDA (offline):", pda.toBase58());
```

This is synchronous and requires no RPC call.

## Storing the P-256 key securely

The `p256PrivateKey` is a 32-byte value that determines your on-chain identity. Store it with the same care as a wallet private key.

| Environment | Recommended approach |
|---|---|
| Local dev / CI | File at `0o600` permissions (the tutorial default) |
| Docker / Kubernetes | Kubernetes Secret or Docker secret mounted as a file |
| AWS | AWS Secrets Manager — store as a base64 or hex string |
| GCP | GCP Secret Manager — store as a hex string |
| HashiCorp Vault | `secret/` KV path — store as a hex string |

Add `agent-identity.json` to `.gitignore` immediately:

```bash
echo "agent-identity.json" >> .gitignore
```

## Common errors

| Error | Cause | Fix |
|---|---|---|
| `Transaction simulation failed: invalid program id` | Secp256r1 precompile (SIMD-48) is not active on this cluster | Use devnet or a localnet running Agave 2.0+ |
| `sendAndConfirmTransaction` times out | Devnet congestion or insufficient fee payer balance | Airdrop more SOL; retry — the call is idempotent |
| Different `agentWallet` on each restart | A new P-256 key generated each run | Persist `p256PrivateKey` and reload it as shown in Step 6 |
| `EscrowAgentWalletRequiredError` from `createPact` | `agentWallet` not passed to `createHoldfastClient` | Pass the PDA from the registration result |
| `InvalidAgentKey` (on-chain) | P-256 key coordinates are zero | Should not occur with SDK-generated keys; file a bug if seen |
| `AttestationChallengeMismatch` (on-chain) | Signature preimage hash mismatch | SDK version mismatch — ensure `@holdfastprotocol/sdk@devnet` |
| `DirectInvocationRequired` (on-chain) | `register_agent_wallet` called via CPI | Must be invoked as a top-level instruction, not through another program |

## What is SIMD-48?

The registration transaction includes a `Secp256r1Program` instruction — a Solana-native precompile activated by [SIMD-48](https://github.com/solana-foundation/solana-improvement-documents/pull/48) that verifies P-256 (secp256r1) signatures at consensus level. This lets the `register_agent_wallet` instruction confirm key ownership on-chain without trusting the submitter.

Devnet has had SIMD-48 active since early 2025. Mainnet-beta activation is tracked separately; Holdfast Protocol's mainnet launch is gated on both that activation and the external security audit.
