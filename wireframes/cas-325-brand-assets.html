Deployment note — accent color reconciliation

The issue description references #10b981 (emerald) as the primary Holdfast accent. This predates the brand guide established in CAS-245.

The canonical accent color is #2D8CFF (trust blue), as specified in design-tokens.html and all subsequent wireframes.

All assets in this package use #2D8CFF. The issue description's "emerald" reference is treated as a pre-brand artefact and disregarded.

PNG rasterization — recommended commands
# Install Inkscape CLI (or use ImageMagick / Squoosh)

# Rasterize favicon SVG to all required PNG sizes
inkscape public/favicon.svg --export-png=public/favicon-16x16.png --export-width=16 --export-height=16
inkscape public/favicon.svg --export-png=public/favicon-32x32.png --export-width=32 --export-height=32
inkscape public/favicon.svg --export-png=public/apple-touch-icon.png --export-width=180 --export-height=180
inkscape public/favicon.svg --export-png=public/android-chrome-192x192.png --export-width=192 --export-height=192
inkscape public/favicon.svg --export-png=public/android-chrome-512x512.png --export-width=512 --export-height=512

# Generate favicon.ico (multi-resolution, requires ImageMagick)
magick public/favicon-16x16.png public/favicon-32x32.png public/favicon.ico

# Screenshot OG image HTML → PNG (requires Playwright or Puppeteer)
npx playwright screenshot --browser=chromium --viewport-size=1200,630 \
  wireframes/cas-325-og-image.html public/og-image.png
06 Implementation

layout.tsx Integration

Wire all assets into app/layout.tsx. The metadata object needs favicon icons, OG image, and Twitter image fields. The <head> block needs the webmanifest link and apple-touch-icon.

app/layout.tsx — complete updated metadata + head
export const metadata: Metadata = {
  title: {
    default: SITE_TITLE,
    template: "%s | Holdfast",
  },
  description: SITE_DESC,
  // ... existing fields ...

  // ── Icons ──────────────────────────────────────
  icons: {
    icon: [
      { url: "/favicon.svg", type: "image/svg+xml" },
      { url: "/favicon-32x32.png", sizes: "32x32", type: "image/png" },
      { url: "/favicon-16x16.png", sizes: "16x16", type: "image/png" },
    ],
    apple: [{ url: "/apple-touch-icon.png", sizes: "180x180" }],
    other: [{ rel: "mask-icon", url: "/favicon.svg", color: "#2D8CFF" }],
  },

  // ── OpenGraph ──────────────────────────────────
  openGraph: {
    type: "website",
    locale: "en_US",
    url: SITE_URL,
    siteName: "Holdfast",
    title: SITE_TITLE,
    description: SITE_DESC,
    images: [
      {
        url: "/og-image.png",
        width: 1200,
        height: 630,
        alt: "Holdfast Protocol — Trust infrastructure for the autonomous agent economy",
        type: "image/png",
      },
    ],
  },

  // ── Twitter ────────────────────────────────────
  twitter: {
    card: "summary_large_image",
    title: SITE_TITLE,
    description: SITE_DESC,
    images: ["/og-image.png"],
  },

  // ── Manifest ───────────────────────────────────
  manifest: "/site.webmanifest",
};

// In the <head> block of RootLayout, add:
<link rel="manifest" href="/site.webmanifest" />
<meta name="theme-color" content="#0D1117" />
<meta name="msapplication-TileColor" content="#0D1117" />
Next.js note: When metadata.manifest and metadata.icons are set, Next.js 14+ automatically injects the correct <link> tags. The manual <head> additions above cover theme-color (browser chrome tint) and MS tile color, which are not exposed through the Metadata API.
Hardline Protocol — brand differentiation note

Per the issue scope: Hardline Protocol should feel related but distinct. The shield motif and Inter typeface carry over. The recommended accent swap is #9F6BFF (purple — "authority, enforcement, immovable"), already established in design-tokens.html as --accent-hardline.

Hardline favicon would use the same shield silhouette filled with #9F6BFF on the same #0D1117 background. No separate geometry needed — only the fill color changes.

This is out of scope for CAS-325 but deferred as a follow-up ticket when Hardline Protocol front-end work begins.
