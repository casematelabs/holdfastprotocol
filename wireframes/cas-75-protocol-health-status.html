§0  Overview

Three sub-components — Oracle Node Status, Indexer Sync Lag, and On-Chain Events Feed — sit in a single horizontal strip at the top of the P1 operator dashboard. §1  Oracle Node Status
Three States

The oracle component surfaces node heartbeat health as a color-coded pill, a relative timestamp, 24h uptime, and a 8-bar response-time sparkline. A contextual banner below the metrics appears only in degraded and down states.

§3  On-Chain Events Feed
Loaded · Loading · Empty

The events feed shows the last 10 protocol instructions with an iconographic event type, relative timestamp, and truncated txn hash linking to Solana Explorer. A refresh countdown is always visible in the panel header. Rows fade in on new events — no full panel reload.

Icon design intent Icons use Unicode glyphs as placeholders in wireframe. Final implementation should use a consistent SVG icon set (e.g. Phosphor or custom). Each event type gets a unique hue: blue = identity events (init, register), green = resolution events (update_rep, release_pact), purple = contract events (create_pact), amber = financial events (deposit_escrow), red = dispute events (open_dispute).

§4  Interaction & Behavior
States, Transitions & Data Contracts

Frontend implementation notes for engineers. All three components are read-only panels — no user input. Data is polled; no WebSocket required for MVP.

Poll Intervals
ComponentPoll intervalEndpoint
Oracle status10s/api/oracle/status
Indexer sync lag5s/api/indexer/status
Events feed30s (countdown)/api/events/recent?limit=10
Timestamps10s client tickNo refetch. DOM update only.
Transition Rules
TransitionBehavior
Healthy → Degraded Pill color transitions immediately. Banner slides in below oracle body with 200ms ease-out. No page flash.
Degraded → Down Pill updates. Banner copy swaps. Border-top color transitions from amber → red.
Down → Healthy Banner slides out (200ms). Pill returns green. Sparkline resets from current polling window — no history replay.
Events: new event arrives New row slides in at top (translateY -44px → 0, 150ms ease-out). 10th row fades out simultaneously.
Indexer lag threshold cross Numeral color transitions via CSS color: interpolate where supported. Threshold legend chip updates instantly.
Error / stale data handling If any poll fails: component enters a stale state — data remains visible but a small "⚠ data stale" badge appears in the panel header next to the live dot. The live dot dims to 30% opacity. After 3 consecutive failures, the component enters an error state that matches the "Down" oracle variant styling. A "Retry" button replaces the refresh countdown in the events feed.
