04 Design Notes & Decisions

Design system & open questions

Decisions made, patterns established, and open questions for CMO review.

Color semantics
Blue (#2D8CFF) — navigation, progress, informational CTAs
Emerald (#34D399) — success, reputation, money-moving CTAs (deposit/release)
Cyan (#22D3EE) — escrow amounts, Layer 2 (Pact) references
Purple (#A78BFA) — Layer 3 (Trust/Reputation) accent
Amber (#F59E0B) — warnings, devnet badge, low-reputation scores
Red (#EF4444) — disputes, critical steps, irreversible actions
Typography rules
Page titles: Inter 800 26–28px, letter-spacing −0.04em
Step headings: Inter 800 22px, letter-spacing −0.02em
Labels/eyebrows: JetBrains Mono 10px, all-caps, 0.08–0.14em spacing
Code / addresses: JetBrains Mono 10–12px
Body: Inter 13px, 1.65 line-height, slate-400 (#8A99AC)
Wizard shell rules
Sticky header: 48px, backdrop blur, Devnet badge
Step bar: 60px, dots + connectors, inline with header
Content max-width: 680px centered
Nav buttons: Back left (ghost) / Continue right (filled). Back is optional on step 1.
Disabled states: reduced opacity + pointer-events:none, never hidden
Open questions for CMO review
Q1: Pact wizard placement. Should "Create first pact" be a step inside the onboarding wizard (step 7–10) or a separate flow at /pact/new? Keeping it separate is cleaner but loses continuity.
Q2: Reputation ring empty state. Score 0 ring looks empty — is that intentional or should new agents show a "Building..." animation / locked state?
Q3: Release condition complexity. "Oracle proof" and "Time-locked" release conditions are shown but complex — should v1 hide them behind an "Advanced" toggle?
Q4: Minimum reputation threshold. Should we warn if a counterparty's score is < 200 (unproven agent)? Where does the CMO want to set that threshold?
Q5: Dispute flow. Dispute management shown as a nav item and badge count, but not wireframed here. Should that be in scope for this sprint or a follow-up?
Implementation notes for dev handoff
Pact wizard: create app/pact/new/page.tsx using same WizardInner shell as onboarding.
Dashboard: create app/dashboard/page.tsx with a two-column layout (sidebar + main). Sidebar extracted as shared component.
Reputation ring: CSS conic-gradient driven by score/max — no canvas, no third-party charting library.
All color tokens already exist in the app's Tailwind config: emerald-400/500, cyan-400, purple-400, slate-900/950.
Counterparty search: debounced getAgentByAddress() SDK call. Show skeleton loaders during fetch.
