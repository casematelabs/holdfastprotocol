§5 Component Spec Reference
Token & Component Inventory
Design tokens, interaction states, and component boundary definitions for implementation handoff.
State Colour Tokens
Open --warning / --warning-dim / --warning-border
Arbiter Assigned --purple / --purple-dim / --purple-border
Resolved --success / --success-dim / --success-border
Expired --text-tertiary (muted)
Stuck (H-2) --danger / --danger-dim / --danger-border
New token: --purple Add to design system: #A855F7, dim rgba(168,85,247,0.10), border rgba(168,85,247,0.22). Used exclusively for Arbiter state to distinguish it from generic "in progress" (blue) and "warning" (amber).
Component Boundaries
DisputeCard
Props: disputeId, state, deadlineSecs, parties, pdaAddress. Renders state machine breadcrumb, deadline row, parties grid. No CTA — actions are in DisputeActionPanel.
DisputeActionPanel
Props: role (initiator|counterparty|arbiter), dispute, isWindowOpen, escrowAmountSol. Renders BPS slider + split preview + submit CTA. Locked state when isWindowOpen=false.
DisputeActivityFeed
Props: events[]. Event types: dispute_opened, arbiter_assigned, resolution_submitted, expired. Each event: timestamp, actorAddress, txnSignature.
DisputeConfirmModal
Props: splitBps, initiatorAddr, counterpartyAddr, escrowSol, onConfirm, onCancel. Inline overlay or modal. Shown only when user clicks Submit Resolution.
DisputeEmptyState
Shown when disputes.length === 0. No props. Static informational surface.
DisputeStuckBanner
Shown when state=stuck (H-2 gap active). Props: cas106Url. Renders H-2 protocol gap warning. References CAS-106. Remove when CAS-106 ships.
Interaction & Behaviour Spec
BPS input: accepts 0–10000 integer. Slider and numeric input are synced bidirectionally. Preview updates on change.
Submit CTA: disabled if isWindowOpen=false OR role≠arbiter. On click: show DisputeConfirmModal. On confirm: call resolve_dispute SDK method. Transition card to "Resolved" state on txn confirm.
Deadline countdown: live countdown from dispute_deadline_secs. Source: on-chain, not system clock. When it reaches 00:00:00, switch deadline row to "expired" styling immediately.
Address display: truncate to first 4 + last 4 chars with "…" separator. Copy button copies full base58 address. Tooltip on hover shows full address.
Txn links: open Solana Explorer in new tab. Include rel="noopener noreferrer".
Layout & Responsive Breakpoints
≥ 1024px (desktop): Two-column dispute-layout. Left: dispute card + feed (1fr). Right: action panel (360px fixed).
768px–1023px (tablet): Stack to single column. Action panel moves below dispute card. Parties grid collapses to 2 + 1 grid or single column.
≤ 767px (mobile): Full single column. BPS slider accessible but numeric input is primary input. Confirm modal is full-screen sheet.
Navigation: Dispute tab in P1 dashboard nav. Badge shows count of open disputes. Badge hidden (or shows 0 with no alert styling) when empty.
