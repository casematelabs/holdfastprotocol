Holdfast Protocol Dashboard UI v1 2026-04-22 · v1.0
Extends CAS-72. These components are additions to the existing Reputation page. They slot below the existing tier-badge grid, sparkline, stats row, and pre-flight indicator. Section 01 adds a trigger button adjacent to the score display. Sections 02–04 appear below existing content in order. On-chain threshold persistence (Section 04) depends on CAS-121 SDK/program wiring; shown here with dependency note.
01 — Score Explanation Modal
Score Explanation Modal
A "Why is my score X?" button sits inline with the score number in the existing sparkline card header. On click, it opens an accessible modal (role="dialog", aria-modal="true") breaking down each factor's contribution. Keyboard-navigable: Tab cycles focus within modal, Escape closes it. Focus returns to trigger button on close.

05 — Interaction Notes
Interaction & Accessibility Notes
Behavior contract for engineering handoff.
Score Modal
  • Trigger: "Why is my score X?" button in sparkline-card header. Button label is dynamic: updates with oracle score.
  • On open: set aria-expanded=true on trigger, render overlay + modal, trap focus inside modal.
  • Close: × button, Escape key, or overlay click. Restore focus to trigger button.
  • Focus order: Close button → Factor rows (read-only) → Methodology link.
  • Score data loaded from oracle state — same source as sparkline-card.
Tier Upgrade Path
  • Show only when operator is below Platinum (i.e. has a next tier). Hide entirely at Platinum.
  • At Platinum: replace with "You've reached the top tier — maintain your score to retain standing."
  • Metric selection: show top 2–3 factors with largest gap between current value and tier threshold, determined at render time from oracle data.
  • Progress fill: clamp at 95% if operator has reached threshold (to avoid 100% fill before tier actually upgrades via oracle).
History Table
  • Collapsed by default on first page load. Expand state persisted: localStorage['rep-history-expanded'].
  • Table rows: limited to 30 most recent events. Pagination not required for v1.
  • Oracle Update ID: truncated address (first 6 + last 4 chars). Full address in tooltip. Link opens chain explorer in new tab.
  • Collapsible animation: height 0 → auto with CSS transition, not display:none (to avoid reflow jump).
On-Chain Thresholds
  • On "Save to Chain" click: validate inputs → show "Signing…" state (spinner + pulse badge) → send tx → on success show "Chain" green badge + last-saved date; on failure restore "Local only" badge + toast error.
  • "Save local" always available regardless of CAS-121 status. Writes to localStorage immediately.
  • Badge on initial load: read from program account if CAS-121 wired; fall back to localStorage. If both exist, prefer on-chain.
  • Disabled "Save to Chain" (pre-CAS-121): opacity 0.35, cursor not-allowed, tooltip on hover.
