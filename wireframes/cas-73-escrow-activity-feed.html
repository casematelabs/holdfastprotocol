01 — PactStatusPill
Status Pills
Four discrete pact lifecycle states. Disputed uses an animated dot — the only pulsing element in the system — to signal that operator attention is required. Color selection avoids clash with tier badge palette. All pills use tabular-caps for optical alignment in table columns.

Design note Disputed is the only state that pulses. This signals live financial urgency without being disruptive on non-dispute views. Pills are uppercase, 9–10px, to avoid competing with amounts/addresses in the same table row. Cancelled uses a desaturated style — it is a resolved terminal state, not an alert.

03 — DisputePanel + DisputeCard
Open Disputes Panel
Disputes are financially critical — an open dispute with an approaching time-lock means funds may be unrecoverable after deadline. The panel sits above the pact table in the dashboard layout and is hidden when dispute count is zero. Three urgency tiers shift color and add a pulse indicator as the deadline approaches. The countdown timer uses tabular monospace for stability.

Urgency thresholds Tier 1 (>48h): amber — dispute is open but not time-critical. Tier 2 (<24h): orange + pulse — operator should act. Tier 3 (<6h): red + pulse + CRITICAL — highest urgency, review immediately. The countdown updates every second via a client-side interval. Font variant tabular-nums prevents layout shift as digits change. Panel is fully hidden (height: 0, no DOM space) when dispute count is zero — it must not leave a ghost gap in the non-dispute view.

06 — Panel States
Loading, Error, and Empty States
Each of the three panels (Active Pacts, Open Disputes, Completed Feed) has three non-data states. Skeletons mirror the final content shape. Error states include retry. Empty states are contextually differentiated — Active Pacts has a CTA, Completed Feed does not, Disputes panel is fully removed from DOM.

State machine per panel Each panel independently tracks: idle → loading → { data | empty | error }. Transitions: idle→loading on mount; loading→data on success; loading→error on failure; data→loading on retry. Polling interval: 30s for disputes (time-sensitive), 60s for active pacts, on-demand for completed feed.
07 — Layout Overview
Dashboard Panel Composition
The three panels are arranged vertically: Disputes (conditionally) → Active Pacts → Completed Feed. Disputes panel occupies the top position when active, ensuring financial urgency is immediately visible. On narrow viewports (<900px), table condition column collapses to icon-only.
Open Disputes 2

Active Pacts
↑ PactTable — primary content area, full width
Completed Pacts

Stacking order rationale Disputes above pact table — an open dispute requires immediate action and should never be obscured by a full table of active pacts. Completed feed below fold is intentional: it is historical context, not an action surface.

09 — Design Tokens
Color & Status Tokens
All status colors are drawn from the shared Holdfast Protocol token set established in CAS-72. The urgency-orange token (--urgent) is new and unique to the dispute system.

pulse (1.4s) Disputed pill dot, Tier 2+3 urgency badge
shimmer (1.5s) All skeleton loading states
countdown (1s interval) setInterval, cleared on unmount
New token: --urgent (#F97316) Orange sits between amber (--warning) and red (--danger) to provide a perceptible 3-step urgency ramp. Without this middle step the visual jump from yellow to red is abrupt and may feel alarming even for 23h-remaining disputes that require attention but not panic. Added to the Holdfast Protocol token file as a first-class token.
